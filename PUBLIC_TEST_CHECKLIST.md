# StarX 公测反馈清单

这份清单适合公开测试用户按项回归。没有时间全测时，至少优先覆盖题库源、签到辅助和考试搜题三大块。

## 提交反馈前

1. 先升级到最新公开版本。
2. 先确认问题能稳定复现，或至少记录大概复现概率。
3. 尽量附上截图、录屏或日志。
4. 建议通过 issue 模板提交，而不是直接发一句“不能用”。

## 重点测试项

### 1. 动态题库源

- 新增一个自定义题库源。
- 删除一个自定义题库源。
- 保存配置后重新打开设置页，确认数据没有丢失。
- 检查启用状态、名称、地址、Token 是否正确回显。

### 2. 协议类型切换

- LemTk 兼容源是否能正常查询。
- tikuAdapter 如果填根地址，是否在手动切换到 tikuAdapter 后仍能正常查询。
- ZXSeek / Wkexam 预置源是否能通过快捷填充后正常使用。
- 保存后再次进入设置页，协议类型是否仍保持正确。

### 3. H5 二维码签到

- 页面内有 rcode 时，是否能自动识别并填充。
- 手动填写 H5 二维码参数后，是否能正常触发签到。
- 页面模式识别是否正确，没有误判成其他签到类型。

### 4. 原生二维码签到

- 如果填写的是 signId/time JSON 或带 signId/time 的 URL，是否能直接注入原生扫码流程。
- 如果填写的是只适用于 H5 的 rcode，状态提示是否明确，不应假装原生可用。

### 5. 拍照签到上传

- 拍照签到页是否能在正确时机自动注入图片。
- 非签到页面是否不会误触发图片上传。
- 不同上传入口下是否都能稳定工作。

### 6. 定位签到

- 自动抓取签到目标区域是否有效。
- 手动定位保存后是否优先使用手动坐标。
- 清空手动定位后是否能回退到自动抓取。

### 7. 考试搜题

- 单选、多选、判断、填空是否都能正常识别。
- 有选项时是否优先返回匹配后的答案。
- 题库未命中时，AI 兜底是否按预期生效。
- 不同作业页、考试页、 iframe 页面是否都能触发搜题。

## 建议随 issue 一起提供的信息

- StarX 版本
- 学习通版本
- Android 版本 / 机型 / LSPosed 版本
- 页面入口与页面类型
- 复现步骤
- 期望结果与实际结果
- 复现后的日志

## 日志建议

复现完成后可导出：

```powershell
adb logcat -d -s StarX:D *:S
```

如果问题和 WebView、签到或搜题相关，截图和录屏通常比一句描述更有用。
